INICIANDO CON GIT

GIT es un control de versiones que permite a equipos de desarrolladores trabajar de manera remota sobre un mismo proyecto. permite hacer contribuciones de c√≥digo de manera local y luego replicarlos a un repositorio remoto.

PRINCIPALES COMANDOS CON GIT

  • git init:Funciona para inicializar un proyecto en git.
  • git config --global user.email "email":Para identificar y configurar globalmente el email con el repositorio local.
  • git config --global user.name "name":Para identificar y configurar globalmente el nombre de usuario con el repositorio local.
  • git config user.email :Verificar el email agregado.
  • git config user.name :Verificar el nombre de usuario agregado.
  • git add . :Funciona para agregar todos los archivos modificados y nuevos en el repositorio local de git.
  • git status:Permite verificar el estado del repositorio local.
  • git commit -m "comentario":Permite guardar los cambios realizados en el repositorio local con un mensaje resumido.
  • git log:muestra el historial de cambios en el repositorio local.
  • git branch:Muestra la rama del repositorio local.
  • git checkout -b nombre_rama:cambiar el nombre de la rama en un repositorio local.
  • git checkout -- .: deshacer los cambios en un archivo no confirmado.
